# Memorando Interno — MI-2025-002

**Política de Senhas e Gestão de Acesso aos Sistemas**

## 1. Objetivo
Estabelecer diretrizes para a criação, atualização e uso de senhas na Prefeitura Municipal de Hortolândia, com foco na **proteção dos dados públicos**, **continuidade operacional** e **conformidade com a Lei Geral de Proteção de Dados (Lei nº 13.709/2018)**.

## 2. Diretrizes Gerais

- Cada servidor é **responsável** pela segurança da senha que utiliza.
- Senhas são **pessoais e intransferíveis**; é proibido compartilhá-las com colegas, estagiários ou terceiros.
- O uso de **senhas iguais** para vários sistemas deve ser evitado, reduzindo riscos de invasão.
- Senhas devem ser alteradas sempre que houver **suspeita de vazamento**, troca de setor ou desligamento de funções.

## 3. Requisitos Mínimos para Senhas
As senhas corporativas devem conter:

- Pelo menos **8 caracteres**.
- Ao menos **1 letra maiúscula**, **1 letra minúscula**, **1 número** e **1 caractere especial**.
- Não devem conter: nomes próprios, datas de aniversário, números sequenciais ou informações óbvias.

Exemplo recomendado:
C0nex@o_Segur@nç4

## 4. Troca Periódica e Bloqueio
- Recomenda-se a **troca de senha a cada 180 dias**.
- Após **5 tentativas incorretas**, o acesso poderá ser bloqueado temporariamente como medida de proteção.
- O desbloqueio deve ser solicitado via **chamado oficial** ao DITI.

## 5. Acessos e Permissões
| Situação | Procedimento |
|--------|-------------|
| Servidor inicia atividade | Solicitar criação de usuário via chamado |
| Mudança de setor | Solicitar ajuste de perfil de acesso |
| Saída / Exoneração | Acesso deve ser revogado **imediatamente** |

*Observação:* A política segue o princípio de **menor privilégio**: cada usuário tem apenas os acessos necessários para suas funções.

## 6. Responsabilidades
| Unidade | Responsabilidade |
|--------|-----------------|
| **DITI** | Definir políticas, orientar servidores, auditar acessos e responder incidentes |
| **Secretarias** | Comunicar movimentações de pessoal e garantir cumprimento das regras |
| **Servidores** | Manter a confidencialidade e cuidado com suas credenciais |
